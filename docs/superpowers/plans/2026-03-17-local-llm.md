# Local LLM Integration Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add LocalAI as always-on local AI inference to Bloom OS with the omnicoder-9b model bundled, registered as Pi's default provider.

**Architecture:** A NixOS module (`bloom-llm.nix`) defines a systemd service that seeds and runs LocalAI on port 11435. A Pi extension (`localai`) registers LocalAI as a provider at session load. The wizard's `step_ai` is updated to write LocalAI as the default provider/model into Pi's settings.

**Tech Stack:** Nix/NixOS, `pkgs.local-ai` (nixpkgs-unstable), TypeScript/Vitest, bash

- [ ] **Step 2.1: Create `core/os/modules/llm.nix`**

  ```nix
  # core/os/modules/llm.nix
  { pkgs, lib, ... }:

  let
    modelFileName = "omnicoder-9b-q4_k_m.gguf";
    model = pkgs.fetchurl {
      url    = "<URL from Task 1>";
      sha256 = "<hash from Task 1>";
      name   = modelFileName;
    };
  in
  {
    systemd.services.localai = {
      description = "Bloom Local AI Inference (LocalAI)";
      after    = [ "network.target" ];
      wantedBy = [ "multi-user.target" ];

      serviceConfig = {
        Type             = "simple";
        ExecStartPre     = "${pkgs.writeShellScript ''localai-seed-model'' ''
          dest=/var/lib/localai/models/${modelFileName}
          if [ ! -f "$dest" ]; then
            install -m 644 ${model} "$dest"
          fi
        ''}";
        ExecStart        = "${pkgs.local-ai}/bin/local-ai run --address 0.0.0.0:11435 --models-path /var/lib/localai/models";
        Restart          = "on-failure";
        RestartSec       = 5;
        DynamicUser      = true;
        StateDirectory   = "localai localai/models";
        WorkingDirectory = "/var/lib/localai";
      };
    };
  }
  ```

  Key points:
  - `''..''` (Nix multiline strings) used for both `writeShellScript` args — avoids nested double-quote parse errors
  - `"${pkgs.writeShellScript ...}"` — the outer `"..."` coerces the store path to a string, required by `ExecStartPre`
  - `StateDirectory = "localai localai/models"` — systemd creates BOTH directories and chowns them to `DynamicUser` before `ExecStartPre` runs
  - `ExecStartPre` is idempotent: skips copy if model file already present

  # Local LLM (LocalAI)

  LocalAI runs on every Bloom OS instance and is always available at boot. It provides an OpenAI-compatible API for four modalities: LLM, speech-to-text, text-to-speech, and image generation.

  ## Service

  LocalAI is a system service managed by systemd:

  ```bash
  systemctl status localai          # check status
  sudo systemctl restart localai    # restart if needed
  ```

  API base URL: `http://localhost:11435/v1`

  List loaded models:
  ```bash
  curl http://localhost:11435/v1/models
  ```

  ## Default Model

  `omnicoder-9b-q4_k_m` is pre-loaded at boot. No download needed.

  ## Endpoints

  | Modality | Endpoint | Notes |
  |----------|----------|-------|
  | LLM | `POST /v1/chat/completions` | OpenAI-compatible chat |
  | STT | `POST /v1/audio/transcriptions` | Whisper — transcribe audio files |
  | TTS | `POST /v1/audio/speech` | Text to speech |
  | Image | `POST /v1/images/generations` | Image generation |

  ## LLM Example

  ```bash
  curl http://localhost:11435/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{
      "model": "omnicoder-9b-q4_k_m",
      "messages": [{"role": "user", "content": "Hello"}]
    }'
  ```

  ## STT Example (Whisper)

  ```bash
  curl http://localhost:11435/v1/audio/transcriptions \
    -F "file=@audio.mp3" \
    -F "model=whisper-1"
  ```

  Note: Whisper requires a whisper model to be loaded. Download a whisper model from the LocalAI model gallery and drop it in `/var/lib/localai/models/`.

  ## Adding More Models

  Drop any GGUF file into `/var/lib/localai/models/` and restart the service:

  ```bash
  sudo cp mymodel.gguf /var/lib/localai/models/
  sudo systemctl restart localai
  ```

  ## When to Use Local vs Cloud

  **Prefer local:**
  - Offline or air-gapped operation
  - Privacy-sensitive tasks (nothing leaves the device)
  - Bulk processing (no rate limits)
  - Audio transcription or synthesis

  **Prefer cloud Claude:**
  - Tasks requiring strong reasoning or very large context
  ```
